## Workflow für jeden Roman

### Phase 1: Spezifikation (5%)
1. Neues Projekt erstellen
2. Alle Spezifikationsfelder ausfüllen
3. KI-Titelvorschläge generieren
4. Genre-Template als Basis verwenden

### Phase 2: Brainstorming (10%)
1. Kernidee entwickeln
2. Fragen beantworten
3. Fundamente extrahieren
4. Plot-Ideen sammeln

### Phase 3: Plot (15%)
1. Story Beats definieren (Hook → Resolution)
2. Alle 9 Beats ausarbeiten
3. Subplots hinzufügen
4. Spannungskurve prüfen

### Phase 4: Charaktere (20%)
1. Protagonist erstellen (detailliert)
2. Antagonist erstellen
3. Supporting Characters (3-5)
4. Beziehungen definieren
5. Voice DNA analysieren

### Phase 5: Szenen (25%)
1. Szenen für jeden Plot Beat
2. Kapitelzuordnung
3. POV-Verteilung
4. Timeline prüfen

### Phase 6: Schreiben (60%)
1. Kapitel für Kapitel generieren
2. Szenen-Prose mit KI
3. Übergänge schreiben
4. Dialog verfeinern

### Phase 7: Review (15%)
1. KI-Kritik für jedes Kapitel
2. 12-Dimensionen-Analyse
3. Kontinuitätsprüfung
4. Überarbeitung bis Score ≥ 9.5

### Phase 8: Finalisierung (5%)
1. Gesamtbewertung prüfen
2. Export als DOCX
3. Finale Qualitätskontrolle

---

## Qualitätssicherung

### Bewertungsdimensionen (je 1-10)
1. **Pacing** - Tempo und Rhythmus
2. **Voice** - Erzählstimme
3. **Dialogue** - Dialogqualität
4. **Tension** - Spannung
5. **Show/Tell** - Zeigen vs Erzählen
6. **Character** - Charaktertiefe
7. **Setting** - Weltenbau
8. **Plot** - Handlungslogik
9. **Theme** - Thematische Tiefe
10. **Prose** - Sprachqualität
11. **Stakes** - Einsätze/Risiken
12. **Hook** - Aufhänger

### Ziel: Durchschnitt ≥ 9.5

Bei Score < 9.5:
1. Schwächste Dimension identifizieren
2. Kapitel überarbeiten
3. Erneut bewerten
4. Wiederholen bis Ziel erreicht
